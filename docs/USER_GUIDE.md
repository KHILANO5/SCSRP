# SCSRP - User Guide

Welcome to the Smart Campus Service Request Portal. This guide will help you navigate the application whether you are a Student or an Administrator.

## üéì For Students

### 1. Account Creation
*   Navigate to the **Register** page.
*   Select the **Student** tab.
*   Enter your Full Name and Password.
*   **Important**: Use your university email ending in `@student.university.edu`.
*   Click **Create Account**.

### 2. Dashboard Overview
Once logged in, you will see the Student Dashboard:
*   **"New Request" Button**: Top right corner to start a report.
*   **Status Tabs**: Filter your list by "All", "Pending", "In Progress", or "Resolved".
*   **Request Cards**: Each card shows the request title, category, date, and current status badge.

### 3. Creating a Request
1.  Click **New Request**.
2.  **Category**: Select the most relevant area (e.g., Classroom, Hostel, Library).
3.  **Title**: A short summary (e.g., "Broken Chair in Room 301").
4.  **Description**: Detailed information about the issue.
5.  **Image** (Optional): Upload a photo of the issue.
6.  Click **Submit Request**.

### 4. Tracking Status
*   **Pending (Orange)**: Your request has been sent but not yet reviewed.
*   **In Progress (Blue)**: Admin has seen the request and assigned it to a team.
*   **Resolved (Green)**: The issue has been fixed.

### 1. Access
*   Login using your administrative credentials (email ending in `@admin.university.edu`).

### 2. Admin Dashboard
You see a comprehensive view of campus maintenance:
*   **Stats Overview**: Quick count of Total, Pending, and Resolved requests.
*   **Filters**: Sort the list by specific Category or Status.

### 3. Managing Requests
Click on any request card to open the **Details Modal**.
*   **View Details**: See the student's description and uploaded image.
*   **Admin Actions**:
    *   **Mark In Progress**: Click this when you assign a team. You can add an "Assigned To" note (e.g., "Electrical Dept").
    *   **Mark Resolved**: Click this when the work is done.
    *   **Add Notes**: Internal notes for other admins to see (e.g., "Waiting for parts").

### 4. Best Practices
*   Regularly check "Pending" requests to keep the queue moving.
*   Use the "Notes" feature to document delays or part orders.
*   Ensure the status is updated to "Resolved" as soon as work is complete to notify the student.
